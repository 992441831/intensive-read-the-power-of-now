# 精读《当下的力量》· 新篇创作流程

用户调用本 skill 时，可能给出选题（如"痛苦之身与深夜 emo"），也可能只说"继续下一篇"。按以下步骤执行。

系列定位：回到埃克哈特·托利《当下的力量》（通行中译本：曹植译，中信出版社）的概念内核——思维认同、小我、痛苦之身、心理时间、临在、臣服、内在身体——用它们解剖当代人的精神困境（精神内耗、注意力碎片化、焦虑、失眠、刷屏成瘾），同时保持批判性反思，不沦为鸡汤。

## 第 0 步：确定篇目

1. `ls reports/` 确定下一篇编号 NN（最大编号 + 1）。
2. 选题：用户指定则用之；否则从候选池推荐一个并确认：
   - 你不是你的大脑：思维认同与精神内耗（第一章）
   - 痛苦之身：深夜 emo 与情绪反刍（第二章）
   - 心理时间与钟表时间： deadline 焦虑与"等以后就好了"（第三章）
   - 临在与观察者：注意力经济时代的走神（第四章）
   - 内在身体：用身体锚定当下（第六章）
   - 臣服：躺平、摆烂与真正的接纳（第七章/第十章）
   - 开悟的爱情关系：亲密关系中的痛苦之体共振（第八章）
   - 超越幸福与不幸的平和（第九章）

## 第 1 步：写 Markdown 源文件 → `reports/NN-中文标题.md`

先读最近一篇校准文风。固定结构：

1. `# emoji 标题` + 副标题引文（托利原话）+ 元信息行（文本依据注明《当下的力量》章节 + 当天日期）
2. `> [!NOTE]` 核心论点
3. `## 一、引言`（2026 年当代现象切入：刷屏、内耗、失眠、焦虑数据）+ `> [!IMPORTANT]` 问题的提出（先破主流解释——如"内耗是意志力不够""正念就是放松"）
4. `## 二、概念基石`（分小节讲透托利概念，引原书原文 blockquote + 出处行，出处写到章节即可，不编页码；至少一个 `> [!WARNING]` 纠正常见误读——如"不认同思维 ≠ 停止思考""临在 ≠ 放空发呆""臣服 ≠ 躺平认命"）
5. `## 三、一张表：A × B`（如"思维认同 × 临在观察""小我 × 本体"，加粗维度列）
6. `## 四、解剖XX`（开头用 `> [!NOTE]` 写明本篇与系列前篇的递进关系；分 3-4 个小节解剖当代困境，行文自然"呼应第N篇"；首篇无呼应要求）
7. `## 五、超越托利：批判性反思`（表格或分节——警惕把痛苦完全内在归因而回避结构性问题、正念产业化/灵性商品化、"灵性逃避" spiritual bypassing；保持知识诚实）
8. `## 六、实践：可以立刻做的练习`（编号步骤，具体可操作，标明对应概念）
9. `## 七、结论`：核心发现（编号列表）→ 出路（表格）→ `> [!TIP]` 延伸阅读/镜鉴 → 托利的启示（引文）→ `> [!TIP]` 最后的判断（加粗核心句收尾）
10. 末尾分隔线 + 斜体报告生成时间行

要求：概念准确，引文用通行中译本（曹植译）；每篇与系列前篇至少 3 处显式"呼应"（第 01 篇除外）；流行心理学术语（内耗、emo、躺平、松弛感）必须还原为托利的概念范畴再展开。

## 第 2 步：写排版 HTML → `docs/english-slug.html`

**风格必须每篇不同。** 先扫一遍现有 `docs/*.html` 的 `<title>` 和背景，确认已用风格；禁止重复。已用风格登记（新篇完成后追加）：

| 篇 | 文件 | 风格 |
|---|---|---|
| 01 | you-are-not-your-mind | 深夜星空风（深蓝夜空渐变、星光粒子、呼吸动画） |
| 02 | the-pain-body | 禅意水墨风（宣纸底纹、晕染墨点动画、朱砂印章、笔触引文条） |
| 03 | psychological-time | 深夜星空风（沿用第 01 篇视觉体系，用户指定：人生感悟类保持静谧一致的星空风格） |

备选新风格：禅意水墨风、极简白底大字呼吸排版、手写日记/笔记本风、日式侘寂风、日出晨光风、植物生长自然风、纸张禅修手账风——也可自创，与主题契合即可。

HTML 内容与 Markdown 对应转换：`> [!NOTE/IMPORTANT/WARNING/TIP]` → 带标签的提示框组件；blockquote 引文 → 引文组件（保留出处 cite）；表格、练习步骤按新风格设计。单文件、内联 `<style>`、无外部依赖、含响应式 `@media`。

## 第 3 步：同步两处索引

- `docs/index.html`：若不存在，先创建系列导航首页（标题：精读《当下的力量》，含系列简介与卡片列表）；在卡片列表末尾追加卡片（报告编号 NN、标题、一句 desc、3 个 tag、英文 slug 链接）
- `README.md`：若为空，先按系列骨架创建（系列简介、📖 报告目录表、🌐 在线阅读说明、📁 目录结构树、✍️ 写作约定、📜 许可）；目录表加一行（# / 报告链接 / 核心议题 / 概念依据）；目录结构树两处（reports/ 与 docs/）各加一行

## 第 4 步：提交并推送

```bash
git add reports/NN-*.md docs/<slug>.html docs/index.html README.md .claude/skills/ .claude/settings.local.json
git commit -m "新增第NN篇：<标题>（<风格名> HTML）"  # 附 Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>
git push
# 若 push 连接 github.com 失败，直接重试：
git -c http.proxy=http://127.0.0.1:10808 -c https.proxy=http://127.0.0.1:10808 push
# 10808 不通再依次试 7890 / 7897 / 10809
```

`.claude/settings.local.json` 也在提交范围内（本仓库约定将其纳入版本管理，便于跨机器同步权限配置）。提交前扫一眼其中是否有敏感信息（token、密钥），若有则先剔除。不要提交 `.idea/` 等无关文件。
